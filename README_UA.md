# BCK — Система резервного копіювання

Корпоративна веб-система керування резервним копіюванням з плануванням, моніторингом, підтримкою різних джерел, контролем доступу на основі ролей, аналітикою в реальному часі, мультитенантністю та підтримкою PWA.

## Можливості

- **Керування бекапами** — створення, запуск та моніторинг бекапів баз даних, ВМ, хмарних сервісів, SSH та хостів
- **Планування** — автоматичні бекапи за cron-розкладом з відстеженням `lastRunAt`
- **Відновлення** — відновлення з будь-якої точки бекапу одним кліком
- **Багато джерел** — MySQL, PostgreSQL, Oracle, MSSQL, MongoDB, Redis; VMware, Hyper-V; AWS S3, GCS, Azure Blob; MinIO (S3-сумісний); SSH віддалений; каталоги Linux/Windows
- **Аналітика дашборду** — графіки в реальному часі (area, pie), індикатори сховища, стрічка активності, розподіл рушіїв та статусів
- **Репозиторії та S3 версіонування** — перегляд усіх бекапів, використання диску, завантаження/видалення; відстеження та відновлення версій S3 об'єктів
- **Мультитенантні організації** — керування кількома організаціями з ізольованими даними, захист типової організації
- **API токени** — генерація токенів з префіксом `bck_` для CI/CD пайплайнів; обмежені дозволи, термін дії, одноразове розкриття
- **LDAP / Active Directory SSO** — аутентифікація користувачів через LDAP/AD з маппінгом груп на ролі
- **Вихідні вебхуки** — підписки на типові події, HMAC-SHA256 підписи, повторні спроби з експоненційною затримкою, журнал доставки
- **Terraform провайдер** — декларативне налаштування бекапів, розкладів та з'єднань через Go-провайдер
- **Prometheus метрики** — `/metrics` ендпоінт з HTTP лічильниками, статистикою бекапів, індикаторами диску, Node.js метриками (опціональний захист `METRICS_TOKEN`)
- **Grafana інтеграція** — готовий дашборд з автопровізією; повний стек Prometheus + Grafana через `docker-compose.full.yml`
- **MinIO S3 сховище** — S3-сумісне об'єктне сховище у повному стеку
- **PWA (Progressive Web App)** — встановлення на мобільний/десктоп; офлайн shell; ярлики; Apple/Android мета-теги
- **Керування користувачами** — контроль доступу на основі ролей з гранулярними дозволами, 2FA (TOTP)
- **Журнал активності** — повний аудит (вхід, вихід, дії користувачів) з відстеженням IP; пагінація та експорт CSV/JSON
- **Політики зберігання** — автоматичне очищення за днями ТА за кількістю копій
- **Щоденне обертання логів** — cron-очищення опівночі; налаштовуваний період зберігання
- **Шифрування** — AES-256 шифрування облікових даних та файлів бекапів
- **Сповіщення** — Email (SMTP), Slack, Discord, Telegram, Webhook
- **Хмарна діагностика** — перевірка з'єднання перед збереженням облікових даних
- **Темна / Світла тема** — сучасний glassmorphism інтерфейс зі збереженням теми
- **Error Boundary** — глобальна React обгортка для обробки помилок з перезавантаженням
- **Обмеження запитів** — 5 спроб входу / 15 хв, 10 запусків бекапів / хв, 200 API викликів / 15 хв

## Швидкий старт

### Linux (однокомандна установка)
```bash
curl -fsSL https://raw.githubusercontent.com/ajjs1ajjs/BCK/master/install.sh | sudo bash
```

### Windows
```batch
git clone https://github.com/ajjs1ajjs/BCK.git
cd BCK
.\install.bat
```

### Або запуск локально
```bash
# Windows:
run-local.bat

# Linux/macOS:
chmod +x run-local.sh && ./run-local.sh
```

Потім відкрити **http://localhost:9000**
Типові облікові дані: `admin` / автогенерований (виводиться при першому запуску) — **змінити після першого входу**

│   ├── queue.js           # Асинхронна черга виконання бекапів
│   ├── webhooks.js        # Сервіс вебхуків та повторних спроб
│   ├── ldap.js            # LDAP/Active Directory SSO сервіс
│   └── s3versions.js      # S3 версіонування об'єктів та відновлення

│   ├── auth.js            # Вхід, вихід, 2FA (TOTP), LDAP/AD аутентифікація
│   ├── backups.js         # CRUD + запуск + завантаження + експорт (CSV/JSON)
│   ├── connections.js     # Керування обліковими даними БД / SSH / Хмари
│   ├── schedules.js       # Cron розклади + lastRunAt + очищення логів

│   ├── auth.js            # JWT аутентифікація + RBAC авторизація
│   ├── rateLimit.js       # Обмежувачі запитів за ендпоінтами

## API Ендпоінти

| Метод  | Шлях                             | Опис                                               |
|--------|----------------------------------|----------------------------------------------------|
| POST   | `/api/login`                     | Аутентифікація (обмеження: 5/15 хв)                |
| POST   | `/api/auth/ldap`                 | LDAP / Active Directory SSO вхід                   |
| POST   | `/api/auth/ldap/test`            | Тест LDAP сервера                                  |
| POST   | `/api/logout`                    | Вихід + аудит                                      |
| GET/POST | `/api/backups`                 | Список (пагінація) / Створення бекапу              |
| GET    | `/api/backups?export=csv\|json`  | Експорт усіх бекапів                               |
| PUT/DELETE | `/api/backups/:id`          | Оновлення / Видалення бекапу                       |
| POST   | `/api/backups/:id/run`           | Запуск бекапу (обмеження: 10/хв)                   |
| GET    | `/api/backups/:id/download`      | Завантаження файлу бекапу                          |
| GET    | `/api/schedules`                 | Список розкладів (з lastRunAt)                     |
| GET/POST/DELETE | `/api/users`            | Керування користувачами                            |
| GET/POST/DELETE | `/api/roles`            | Керування ролями                                   |
| GET    | `/api/logs?page=1&limit=100`     | Перегляд логів (пагінація)                         |
| GET    | `/api/logs?export=csv\|json`     | Експорт логів                                      |
| GET/PUT | `/api/settings`                 | Налаштування системи                               |
| GET    | `/api/stats`                     | Статистика дашборду                                |
| GET    | `/api/health`                    | Перевірка стану                                    |
| GET    | `/api/tools`                     | Перевірка доступності CLI інструментів             |
| POST   | `/api/cloud-credentials/:id/test` | Тест хмарного з'єднання                          |
| GET/POST/DELETE | `/api/webhooks`      | CRUD вебхуків                                      |
| POST   | `/api/webhooks/:id/test`         | Тестовий ping вебхуку                              |
| GET    | `/api/webhooks/:id/deliveries`   | Журнал доставки вебхуку                            |
| GET    | `/api/versions/:backupId`        | Список версій об'єктів S3/MinIO                    |
| POST   | `/api/versions/:backupId/enable` | Увімкнення версіонування на S3 корзині             |
| POST   | `/api/versions/:id/restore`      | Відновлення конкретної версії з S3                 |

## Нотатки безпеки

- Встановіть надійний `ENCRYPTION_KEY` у `.env` (мін. 32 символи, випадковий)
- Встановіть `DEFAULT_ADMIN_PASSWORD` у `.env` або змініть одразу після першого входу
- Увімкніть HTTPS через `SSL_CERT_PATH` та `SSL_KEY_PATH` у `.env`
- Налаштуйте `ALLOWED_IPS` у `.env` для IP фільтрації
